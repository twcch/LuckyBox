# Shipping SOP

## Purpose

Define how LuckyBox handles prize shipment requests, address verification,
tracking updates, returned packages, and defect/exchange cases.

## Standard Flow

1. Review pending shipment requests in the admin shipment queue.
2. Confirm prize ownership, shipment status, recipient name, phone, address, and
   shipping fee/free-shipping coupon state.
3. Pack prizes against the shipment item list; record any missing or damaged
   item before dispatch.
4. Create the parcel with the selected logistics provider.
5. Enter carrier and tracking number in the admin shipment page.
6. Notify the member through the normal customer-support channel if manual
   follow-up is needed.

## Returned Package

- Mark the shipment as returned or exception status.
- Add an internal note with carrier reason, date, and next action.
- Contact the member to confirm address and reshipping cost policy.
- Do not create a replacement shipment without operation owner approval.

## Defect Or Exchange

- Ask the member for photos/video, shipment id, and unboxing context.
- Confirm whether the issue is shipping damage, product defect, wrong prize, or
  disclosure mismatch.
- Preserve evidence in the support case.
- Use the approval center for compensation or point adjustment.
- If multiple members report the same campaign issue, escalate for emergency
  takedown review.

## Done Criteria

- Every dispatched shipment has carrier and tracking number.
- Every exception has an internal note and owner.
- Wallet/prize/shipment state stays consistent after compensation or exchange.
